# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A personal Hugo static site deployed to GitHub Pages at `https://dust115.github.io/dust115/`. Uses the [terminal](https://github.com/panr/hugo-theme-terminal) theme with heavy CSS customization. Content is primarily in Spanish.

## Build & development

```bash
# Local dev server with live reload
hugo server -D

# Build for production (matches CI)
hugo --gc --minify

# New post (uses archetypes/default.md)
hugo new posts/my-post-title.md

# New Códice chapter (its own section)
hugo new codice/codice-XX-title.md
```

Hugo version in CI: **0.152.2** (extended). The theme is a git submodule under `themes/terminal/`.

## Deployment

Pushing to `main` triggers `.github/workflows/hugo.yml`, which builds with Hugo extended and deploys to GitHub Pages automatically. No manual deploy step needed.

## Site structure

- `content/posts/` — blog posts (markdown with TOML/YAML front matter)
- `content/codice/` — chapters of the long-form narrative series "Códice del Polvo" (its own Hugo section; URLs `/codice/<filename>/`)
- `content/codice/_index.md` — landing/index page for the Códice (Era intros are prose; the chapter lists are auto-generated by the `codice-list` shortcode)
- `content/about.md` — about page
- `static/style.css` — all custom CSS overrides for the terminal theme (dark purple palette, Fira Code font)
- `static/` — images (logo, favicon, og-image)
- `layouts/shortcodes/` — custom Hugo shortcodes (see below)
- `layouts/partials/logo.html` — replaces the theme's default text logo with `dust_logo.png`

## Custom shortcodes

| Shortcode | Parameters | Purpose |
|-----------|-----------|---------|
| `{{< listening track="" artist="" album="" >}}` | track, artist, album | Displays a "currently listening" block with accent border |
| `{{< ascii >}}...{{< /ascii >}}` | (inner content) | Wraps braille/ASCII art in a `<pre>` with correct sizing |
| `{{< commit hash="" date="" message="" >}}` | hash, date, message | Renders a git-commit-style footer block |
| `{{< codice-list min="" max="" >}}` | min, max (weight range) | Auto-lists Códice chapters in a `weight` range, sorted by reading order (used by `content/codice/_index.md`) |
| `{{< callout type="" >}}...{{< /callout >}}` | type=info/tip/warning/danger | Highlighted aside (replaces Chirpy `prompt`); colour by type via tokens |
| `{{< command prompt="" >}}...{{< /command >}}` | prompt (default `$`) | Copyable command block (CommandRow style); copy button + decorative prompt |
| `{{< figure src="" alt="" caption="" credit="" >}}` | src, alt, caption, credit | Image with caption + optional credit; resolves `src` as a page-bundle resource if present |
| `{{< badge color="" >}}...{{< /badge >}}` | color (or positional) = accent/purple/green/teal/red/pink/orange/yellow | Inline coloured label (severity/status) |
| `{{< tlp level="" >}}...{{< /tlp >}}` | level (or positional) = clear/green/amber/red | Traffic Light Protocol banner; optional inner note |

Full usage reference with examples: **`docs/shortcodes.md`**.

## Códice del Polvo

The long-form narrative series under `content/posts/codice/`. Structure and conventions beyond the generic site rules:

### Eras

Organized into three eras mapped to narrator phases, reflected in a per-chapter era tag:

| Era | Tag               | Arc                                           |
|-----|-------------------|-----------------------------------------------|
| I   | `zorro-era`       | Chapters 00–06                                |
| II  | `skull-fox-era`   | Chapters 07–16 (plus sub-chapters)            |
| III | `protogen-era`    | Vesper (`v1`–`v5`), Dust (`d1`–`d4`), epílogos |

### Chapter filenames

- Main chapters: `codice-XX-slug.md` (`XX` = zero-padded chapter number, `slug` = kebab-cased subtitle).
- Sub-chapters/vignettes: `codice-XXa-slug.md`, `XXb`, `XXc`, occasionally `XXc2`, `XXc3`.
- Era III: `codice-v1.md`…`v5` (Vesper), `codice-d1.md`…`d4` (Dust), with interstitials like `d1-5`, `d2-5`.
- Epilogues: `codice-epilogo-<slug>.md`.

### Front matter (chapters)

```yaml
---
title: "Códice | X-B — La Olvidada"       # format: Códice | <id> — <subtitle>
date: 2024-04-15T03:00:00-03:00
weight: 2065                               # reading order — see "Reading order" below
draft: false
tags: ["códice", "skull-fox-era"]          # always "códice" + one era tag
series: ["Códice del Polvo"]
summary: "One-line hook shown in listings."
---
```

The 48 chapters migrated from the old `/posts/codice/` location also carry an `aliases: ["/posts/<filename>/"]` so their previous URLs still redirect. New chapters created directly under `content/codice/` do **not** need an alias.

### Reading order (`weight`)

Chapter order is **explicit, via `weight`** — decoupled from `date` (the date means "when published"; `weight` means "where in the book"). Lower `weight` = earlier. Sorting and the prev/next navigation both use it.

**`weight` is an invisible internal sort key, NOT a year or in-story date.** Readers never see it; it is not part of the lore. The story has no specific calendar years — only the three Eras (pasado / presente / futuro). The numeric bands below are just a convention so each Era owns a separate range of sort numbers; they carry no narrative meaning.

| Era | Band (sort range) | Example                               |
|-----|-------------------|---------------------------------------|
| I   | `1000`–`1999`     | `00`→1000, `01`→1010, `03b`→1040 …    |
| II  | `2000`–`2999`     | `07`→2000, `08`→2010 …                |
| III | `3000`–`3999`     | Vesper `v1`→3000 …, Dust, epílogos    |

Gaps of 10 leave room to insert a chapter between two existing ones without renumbering (e.g. between `2010` and `2020`, use `2015`). The navigation partial `layouts/partials/posts_pagination.html` builds prev/next by sorting the Códice pages on `weight`, scoped to the series (it never links out to the blog, and blog posts never link into the Códice).

### Standard opening

Most chapters open with a `{{< listening >}}` block followed by an `{{< ascii >}}` block containing the placeholder `[tu ascii aquí]`. Preserve both unless the user asks otherwise.

### Index page

`content/codice/_index.md` is the Códice landing page (rendered by `layouts/codice/list.html`). It is **semi-automatic**: the Era intros (and the Era III sub-headers **Vesper** / **Dust** / **—**) are hand-written prose, but the chapter lists under each are auto-generated by the `{{< codice-list min="" max="" >}}` shortcode, which lists chapters in a `weight` range sorted by reading order. So the index does **not** drift when you add/remove a chapter.

The `weight` value on each chapter is the single source of truth for order. Current ranges used by the index: Era I `1000–1999`, Era II `2000–2999`, Vesper `3000–3049`, Dust `3050–3139`, epílogos `3140–3999`.

### Adding / removing a chapter

**To add a chapter** (just create the file — the index updates itself):
1. Create `content/codice/codice-<id>-<slug>.md` with the chapter front matter (see above), including:
   - the right era tag (`zorro-era` / `skull-fox-era` / `protogen-era`),
   - a `weight` that places it correctly (pick a value in the gap between its neighbors — see "Reading order"),
   - `series: ["Códice del Polvo"]` (this scopes the prev/next navigation and keeps it out of the blog feed).
2. Build (`hugo --gc --minify`). The chapter appears in the index automatically (in the `codice-list` range its `weight` falls into) and prev/next links recompute. No edit to `_index.md` needed — **unless** it belongs to a brand-new Era or sub-group, in which case add the matching header + a `codice-list` block to `_index.md`.

**To remove a chapter:** delete its file. No renumbering and no index edit needed — the gaps absorb it, and both the index and prev/next recompute from the remaining `weight` values.

**To reorder:** change the `weight` value(s). Nothing else references position.

> Drafts: set `draft: true` on a chapter that isn't ready — it shows in `hugo server -D` but is excluded from production builds, the index and the navigation.

### Voice and language

- **Español neutro** for the main narration. Do not introduce voseo (vos, tenés, sabés), `che`, `pibe`, or other rioplatense markers in narrative prose.
- Individual characters may speak with distinct accents/registers in dialogue — that is intentional and should be preserved. The neutral default applies to narration, not to a character's voice when one has been established.
- Do **not** rewrite the user's prose. Prior attempts (both full and surgical rewrites) failed to preserve the author's voice and were reverted. When asked to help with a chapter, default to structural feedback, diffs, and concrete suggestions — not replacement prose. If you must write prose, mark it as a throwaway draft.
